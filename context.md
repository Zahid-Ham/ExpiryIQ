# ExpiryIQ

## Project Goal

Build a modern enterprise web application that helps organizations manage records with expiry dates and proactively track renewals.

This project is being developed module-by-module.

Never generate the entire project at once.

Every prompt should complete only the requested module while preserving consistency with the existing codebase.

---

## Development Rules

1. Never modify unrelated files.

2. Never redesign existing UI unless requested.

3. Follow existing folder structure.

4. Reuse components whenever possible.

5. Use TypeScript everywhere.

6. Keep components small and reusable.

7. Every feature must compile before moving to the next one.

8. Do not generate placeholder implementations if the actual feature can be built.

9. Always explain which files are created or modified.

10. Never build future modules ahead of schedule.
